Фронтенд

Основні функції:
- Відображення каталогу екзопланет.
- Завантаження даних для аналізу (FITS, CSV, ручне введення).
- Візуалізація результатів аналізу, включаючи графіки та ключові характеристики.

Технології та фреймворки:
- React, TypeScript, Axios, Chart.js, Recharts, React Router.

Взаємодія з користувачем:
- Завантаження файлів (FITS, CSV) або ручне введення даних.
- Фільтрація та пошук у каталозі планет.
- Перегляд та інтерактивна робота з графіками (зум, панорамування, вибір сегментів).

Візуалізації або графіки:
- Графіки світлових кривих (сирі, оброблені, біновані).
- Згорнута фазова крива (phase-folded).
- Список ключових ознак (features) з їх важливістю.
- Список кандидатів у транзити.

Взаємодія з бекендом:
- API-запити (POST) на ендпоінти /predict та /predict_second за допомогою Axios.
- Надсилання файлів (FITS, CSV) або даних з форм.
- Отримання результатів у форматі JSON для подальшої візуалізації.

Особливості дизайну:
- Темна тема з акцентними кольорами.
- Двоколонковий інтерфейс для аналізу (графік + бічна панель).
- Інтерактивні елементи керування для графіків.
- Спрощений процес завантаження даних.

Взаємодія з зовнішніми API:
- Так, фронтенд через проксі-сервер звертається до API NASA Exoplanet Archive (exoplanetarchive.ipac.caltech.edu) для отримання даних каталогу планет.

Бекенд (API)

Основні функції API:
- /predict: Приймає FITS-файли, виконує їх обробку (детренд, ресемплінг), витягує ознаки (tsfresh) та повертає результат класифікації моделлю LightGBM.
- /predict_second: Приймає CSV-файл або дані з форми, виконує класифікацію за другою моделлю.
- /model2/meta: Повертає метадані про ознаки другої моделі.

Параметри та дані:
- /predict: Список FITS-файлів.
- /predict_second: CSV-файл або дані з веб-форми (koi_time0bk, koi_duration та інші).
- Відповіді повертаються у форматі JSON.

Відповіді API:
- Успіх: JSON з результатом (бінарне передбачення, ймовірність, топ-ознаки, дані кривих).
- Помилка: JSON з кодом статусу HTTP (400, 500) та детальним повідомленням про помилку.

Як фронтенд використовує API:
- Фронтенд відправляє дані на аналіз. Отриманий JSON зберігається в localStorage і використовується для відображення сторінки аналізу (/analysis), де дані візуалізуються у вигляді графіків та інформаційних панелей.
